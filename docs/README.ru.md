# Project Planner — Навык AI для OpenCode

[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](LICENSE)

**project-planner** — это навык OpenCode Agent, который обеспечивает профессиональное планирование проекта перед генерацией кода, включая анализ требований, проектирование архитектуры, декомпозицию задач и отслеживание прогресса.

## Возможности

- **Обязательное планирование** — любая задача генерации кода должна пройти планирование, если пользователь явно не пропустит этот шаг
- **Классификация домена** — автоматическое определение типа проекта (Web/CLI/Library/Mobile) с применением соответствующих практик
- **Анализ требований** — 14 измерений для уточнения требований + три уровня приоритета (P0/P1/P2)
- **Оценка рисков** — выявление, оценка и стратегии смягчения 6 категорий рисков
- **Проектирование архитектуры** — ASCII-схема слоёв + модель данных + контракты API
- **Декомпозиция задач** — граф зависимостей + критический путь + оценка времени
- **Отслеживание прогресса** — запись каждого шага + сводка этапов
- **Ноль конфликтов** — не изменяет конфигурационные файлы других плагинов IDE

## Установка

### Способ 1: Глобальная установка (рекомендуется)

```bash
# Windows PowerShell
Copy-Item -Path "path/to/project-planner" -Destination "$env:USERPROFILE\.config\opencode\skills\project-planner" -Recurse

# macOS / Linux
cp -r ./project-planner ~/.config/opencode/skills/project-planner
```

### Способ 2: Установка на уровне проекта

```bash
mkdir -p .opencode/skills
cp -r path/to/project-planner .opencode/skills/project-planner
```

## Конфигурация

```jsonc
{
  "$schema": "https://opencode.ai/config.json",
  "permission": {
    "skill": {
      "project-planner": "allow"
    }
  }
}
```

## Рабочий процесс

Запрос пользователя → определение задачи генерации кода → пропуск планирования? → классификация домена → анализ требований → техническое проектирование → декомпозиция задач → выполнение по задачам

Артефакты: `memory/project-planner/plan.md` → `design.md` → `tasks.md` → `progress.md`

## Совместимость

Не изменяет файлы конфигурации `.vscode/`, `.cursor/`, `.continue/`, `.github/`, `.idea/` и других плагинов.
